# Git Workflow Rules

## Commit Messages (Conventional Commits)

```
<type>(<scope>): <subject>

<body>
```

### Types

| Type | Use for |
|------|---------|
| `feat` | New feature |
| `fix` | Bug fix |
| `docs` | Documentation changes |
| `style` | Code formatting (no logic change) |
| `refactor` | Code restructure (no behavior change) |
| `perf` | Performance improvement |
| `test` | Adding or fixing tests |
| `chore` | Maintenance, dependencies, configs |

### Rules

- **Subject**: max 72 characters, imperative mood ("add" not "added"), no period
- **Body**: required for `feat`, `fix`, `refactor`, `perf`
  - Explain what changed and why
  - Use bullet points for multiple changes
  - Wrap at 72 characters
- English only
- **NEVER add Co-Authored-By lines**
- **NEVER add AI signatures or attribution**
- **NEVER add "Generated by Claude", "Signed-off-by", or similar**

### Examples

```
chore(deps): update angular to v19
```

```
feat(users): add search filter by role

- Add role dropdown to user search form
- Implement filterByRole pipe for client-side filtering
- Update UserService to support role query parameter
```

```
fix(api): resolve null pointer in invoice calculation

Tax-exempt items were included in the taxable subtotal, causing
incorrect tax amounts on invoices with mixed item types.

- Skip exempt items when calculating taxable base
- Add unit tests for mixed taxable/exempt scenarios
```

## Branch Naming

```
<type>/<description-in-kebab-case>
```

Types: `feat/`, `fix/`, `hotfix/`, `refactor/`, `docs/`, `chore/`, `release/`, `test/`

### Rules

- All lowercase
- Hyphens (kebab-case)
- Max ~50 characters
- Descriptive and concise

### Examples

```
feat/user-authentication
fix/invoice-calculation
refactor/pricing-rules
```

## Workflow

- Stage files individually, never `git add .` or `git add -A`
- Don't commit .env, credentials, or secrets
- Don't use `--no-verify` to skip hooks
- Don't force push unless explicitly asked
- Don't amend commits unless explicitly asked
- If pre-commit hooks fail, fix and create NEW commit
